# Симуляция ткани — Position Based Dynamics (WebGPU)

## Запуск

WebGPU не работает при открытии `index.html` прямо с диска (file://) —
нужен http-сервер. Например:

```bash
cd cloth-pbd
python3 -m http.server 8000
```

Открыть `http://localhost:8000/` в свежем **Chrome/Edge 113+** (или любом
браузере с включённым WebGPU).

Управление: ЛКМ + перетаскивание — вращение камеры, колесо мыши — зум,
чекбокс — включение/отключение гравитации.

## Как это устроено

- **Сетка** — квадрат 33×33 вершины, каждая клетка разбита на 2 треугольника
  одной диагональю (как на рисунке в задании).
- **Закреплённые вершины** — 4 угла (`invMass = 0`), отмечены красным.
- **Управляемая вершина** — центральная, отмечена синим. Каждый кадр её
  позиция **напрямую перезаписывается** по закону `y = A·sin(ωt)` в
  compute-шейдере `predict` — это кинематическое перемещение, а не сила,
  как и требуется в задании. От неё расходятся волны по полотну.
- **Метод интегрирования** — semi-implicit Euler + **XPBD** (Extended
  Position Based Dynamics, Macklin/Müller/Chentanez, 2016) для решения
  constraints через compliance (обратную жёсткость) вместо жёсткого
  проецирования — это один из наиболее актуальных алгоритмов в семействе
  PBD (пункт 2 подсказки в задании).
- **Constraints**: структурные (горизонтальные/вертикальные рёбра) и
  диагональные сдвиговые (shear), обеспечивающие треугольникам сопротивление
  скручиванию.
- **Параллельное решение на GPU без атомиков**: рёбра разбиты на 6
  непересекающихся "цветовых" групп (graph coloring) так, что внутри одной
  группы ни одна вершина не встречается дважды. Это позволяет запускать
  6 отдельных compute-دispatch'ей на группу, где потоки безопасно читают и
  пишут в общий storage-буфер позиций без гонок и без atomic-операций
  (которых для float в WGSL нет).
- **Подшаги (substeps)**: на каждый кадр физика считается в 10 маленьких
  подшагах (подход из статьи про XPBD) — это даёт устойчивость ткани при
  простом одноитерационном решателе на подшаг.
- **Рендер**: позиции читаются прямо из storage-буфера по `vertex_index`
  (индексный буфер задаёт треугольники) — без отдельного vertex-buffer.
  Нормаль для простого освещения считается на лету через экранные
  производные (`dpdx`/`dpdy`), без отдельного compute-прохода за нормалями.

## Файлы

- `index.html` — разметка страницы и UI (чекбокс гравитации).
- `style.css` — стили.
- `shaders.js` — все WGSL-шейдеры (`predict`, `solve` (XPBD), `updateVelocity`, `render`).
- `main.js` — построение сетки/раскраска constraints, инициализация WebGPU,
  буферы, пайплайны, камера (орбита мышью), главный цикл симуляции.

## Что можно улучшить дальше

- Добавить bending-constraints (сопротивление изгибу) для более гладкого
  провисания ткани под гравитацией.
- Self-collision (столкновение ткани с самой собой) — в текущей версии не
  реализовано, так как не требуется заданием.
- Вынести число подшагов/жёсткость в UI-слайдеры для интерактивной настройки.
