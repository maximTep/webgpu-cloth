# Ткань на WebGPU

Метод — Extended Position Based Dynamics (compliance-based, Macklin/Muller 2016).

**Сетка**: 33×33 вершины, квадрат 2×2, каждая ячейка режется одной диагональю на 2 треугольника.

**Интегрирование**: semi-implicit Euler на каждом подшаге, дальше XPBD-солвер корректирует позиции. Кадр разбит на 10 подшагов (dt/10) вместо одного большого шага.

**Жёсткость**: задана явно через compliance (обратная жёсткость)
- структурные связи почти нерастяжимые
- диагональные чуть мягче, отвечают за сопротивление скручиванию треугольников

**Раскраска графа**
- горизонтальные рёбра — цвет = i % 2
- вертикальные — цвет = j % 2
- диагональные — цвет = i % 2

Итого 6 групп. Solve-проход на каждой итерации — это 6 последовательных dispatchей, каждый внутри себя полностью параллельный.

**Пины и управляемая вершина**: 4 угла закреплены, центральная вершина двигается по y = A*sin(ωt) — позиция каждый подшаг напрямую перезаписывается в шейдере predict

**Гравитация**: чекбокс, g=-9.8 по Y, действует на все вершины кроме пинов и управляемой

**Стек**: чистый JS+HTML+CSS+WebGPU/WGSL.
